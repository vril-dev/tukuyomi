# tukuyomi v1.1.0 Release Notes

Status: draft
Baseline: v1.0.1

v1.1.0 focuses on DB-backed runtime authority, safer deployment workflows, and
preparation for multi-environment operation.

## Highlights

- Runtime configuration is now DB-authoritative for normalized domains. The
  remaining `config.json` role is DB bootstrap and process-level settings.
- WAF rule assets are managed through DB-backed rule assets, including base
  rule assets and operator-owned additions.
- WAF event persistence now runs asynchronously to reduce proxy-path latency.
- Admin access now uses DB-backed admin users, Argon2id password hashes, signed
  browser sessions, and CSRF protection instead of static admin API keys.
- Sites can carry ACME TLS settings per site, with production/staging selection
  and optional account email.
- Persistent runtime storage now supports local storage and S3 for ACME cache
  material. Azure Blob and GCS are modeled in configuration but intentionally
  rejected in this build until their adapters are implemented.
- Production seed material now lives under `seeds/conf`, so first-import data
  can be adjusted without editing embedded Go code.
- Linux host install gained `make install TARGET=linux-systemd`, and container
  platforms gained `make deploy-render` for generated deployment artifacts.

## Operator Notes

- File-backed runtime fallbacks for policy/rule domains have been removed from
  the active runtime path. Restore or seed data through DB import flows instead
  of relying on `data/conf/rules`, `data/rules`, or `data/geoip`.
- Temporary WAF rule material is staged under `data/tmp` when the runtime needs
  a filesystem view for Coraza.
- `conf/config.json` should remain minimal. It must include storage bootstrap
  settings, but policy and runtime domains should be stored in DB.
- `admin.session_secret` is required for normal operation. Empty or weak
  values fail startup unless `admin.allow_insecure_defaults` is explicitly set
  for local-only testing.
- The first admin owner is bootstrapped from
  `TUKUYOMI_ADMIN_BOOTSTRAP_USERNAME` and `TUKUYOMI_ADMIN_BOOTSTRAP_PASSWORD`.
  Bootstrap is skipped after an admin user already exists.
- Adjust initial DB import material under `seeds/conf`. After import,
  normalized DB rows are the runtime authority.
- `make ui-preview-up` generates an ignored preview-only
  `conf/config.ui-preview.json` with a random session secret, instead of
  weakening or modifying the tracked bootstrap config.
- Provider credentials for persistent storage are read from environment or
  platform identity. They are not stored in JSON config.
- When installing under a login user's home directory, the installer defaults to
  that user and skips `useradd`. System prefixes such as `/opt/tukuyomi` still
  default to the `tukuyomi` system user.
- The first install seed creates a default upstream named `primary`. Operators
  should update it to the real backend endpoint before sending traffic through
  the proxy.

## Admin UI

- Admin UI now starts with a username/password login flow and creates a signed
  session cookie for browser use.
- Rules and rule sets were reorganized around DB-backed assets and Coraza
  ownership.
- Security navigation now separates Coraza-specific controls from request
  controls.
- Logs default to newest-first display, remove the copy action from the list,
  and handle narrow screens with horizontal overflow.
- Cache Rules now reports save/error feedback directly near the relevant
  action area.
- Sites exposes ACME settings for site-level automatic TLS.
- Settings exposes persistent storage configuration, admin session status, and
  operator identity metadata while keeping credentials out of saved JSON.

## Deployment

- `make install TARGET=linux-systemd` builds the binary, creates the runtime
  tree, runs DB migration, imports WAF/CRS assets, seeds the first DB when
  needed, and installs systemd units.
- `make deploy-render` renders deployment artifacts for container image, ECS,
  Kubernetes, and Azure Container Apps targets.
- `make ui-preview-up` builds and syncs the admin UI, generates a secure
  preview bootstrap config, resets/seeds the preview SQLite DB by default, and
  starts both `coraza` and the scheduled-task runner.
- PHP-FPM runtime bundles can be built with `make php-fpm-build RUNTIME=<id>`
  and copied into an installed tree with `make php-fpm-copy`.
- The runtime layout now separates persistent data, temporary material, cache,
  audit output, and DB files.

## Fixes

- Fixed PHP runtime inventory auto-discovery after DB-backed runtime migration.
- Fixed vhost load ordering so PHP runtime cleanup does not hide installed
  runtimes.
- Fixed managed GeoIP update handling so only the country edition is treated as
  the active country DB.
- Fixed cache store runtime directory preparation and mount behavior.
- Fixed install user selection for home-directory prefixes.
- Fixed first-save conflicts on DB-backed policy screens when an installed DB
  has no active version for that policy domain yet.
- Fixed Cache Rules host-scope editing so typing in the host field does not
  remount the editor and drop focus.
- Fixed SMTP notification address validation so missing `host:port` values are
  rejected correctly.
- Fixed scheduled-task smoke coverage to verify both task state and
  stdout/stderr log output across binary, compose, and preview paths.
- Fixed `make ui-preview-up` startup when the tracked bootstrap config has no
  `admin.session_secret`.

## Compatibility

- DB migrations are required before running v1.1.0.
- The admin auth migration adds `admin_users`, `admin_sessions`, and
  `admin_api_tokens`. Existing static admin API key settings are no longer the
  UI login mechanism.
- File seed material may still be used for first import or explicit import
  workflows, but the active runtime should be treated as DB-backed after import.
- Changing `admin.session_secret` invalidates browser sessions and any
  HMAC-peppered admin tokens created with the old secret.
- Azure Blob and GCS persistent storage settings are reserved for future
  adapters and will fail validation if selected in this build.
