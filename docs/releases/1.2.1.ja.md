# tukuyomi v1.2.1 リリースノート

状態: draft
基準バージョン: v1.2.0

v1.2.1 では Gateway runtime model を強化し、Center enrollment workflow に
Gateway config snapshot sync を追加しました。

## 主な変更

- Gateway install は supervised process model に収束します。supervisor が TCP listener
  を所有し、readiness 後に worker を activate します。
- Runtime Apps ownership を supervised Gateway lifecycle 配下へ移し、process-owned な
  app backend が worker activation と揃うようにしました。
- device 承認後、snapshot revision が変わった場合に Gateway が署名付き redacted config
  snapshot を Center へ送信します。
- Center の registered-device 管理 modal から、最新 Gateway config snapshot を download
  できるようにしました。
- UI の長い ETag 表示を `R: cf902153` のような短い revision 表示へ変更しました。
- UI build target は既定で Node 22 npm wrapper を使います。古い system Node がある
  workstation でも Node 18 build に落ちないようにしています。

## 運用上の注意

- `edge.enabled=false` の既存 non-IoT deployment には影響しません。
- IoT / Edge mode でも、Gateway は proxied request ごとに Center へ問い合わせません。
  approval check は local cache を参照し、Center polling と config snapshot sync は
  proxy hot path の外で動きます。
- config snapshot は bounded / redacted です。enrollment token と local device private key
  は Center snapshot に保存しません。
- `make install TARGET=linux-systemd INSTALL_ROLE=gateway` は Gateway install に
  supervised process model を書き込みます。Center install は別の control-plane role のままです。
- legacy single-process Gateway からの初回移行は、listener ownership が変わるため通常の
  service restart が必要です。

## Build / Development

- `tools/npm-node22.sh` を Makefile UI target の既定 `NPM` にしました。
- wrapper は local に Node.js 22.12+ / npm 10+ があればそれを使い、無ければ Docker 経由で
  `node:22-alpine` を使います。
- host に Node 18 が残っていても、`make ui-build-sync` と `make center-ui-build-sync` で
  Node 18 engine warning が出ないようにしました。

## 互換性と移行

- v1.2.1 の起動前に DB migration が必要です。
- Center config snapshot storage は device ごとの最新 snapshot schema を追加します。
- `INSTALL_ROLE` の default は引き続き `gateway` なので、既存 Gateway / Center install
  command は互換です。
