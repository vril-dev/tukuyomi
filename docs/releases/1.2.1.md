# tukuyomi v1.2.1 Release Notes

Status: draft
Baseline: v1.2.0

v1.2.1 hardens the Gateway runtime model and extends the Center enrollment
workflow with Gateway configuration snapshot sync.

## Highlights

- Gateway installs now converge on the supervised process model. The supervisor
  owns TCP listeners and activates the worker after readiness.
- Runtime Apps ownership was moved under the supervised Gateway lifecycle so
  process-owned app backends remain aligned with worker activation.
- Gateway sends a signed redacted configuration snapshot to Center after device
  approval when the snapshot revision changes.
- Center registered-device management can download the latest Gateway config
  snapshot from the device manage modal.
- Long ETag labels in the UI were replaced with short revision labels such as
  `R: cf902153`.
- UI build targets use the Node 22 npm wrapper by default, avoiding accidental
  Node 18 builds on workstations that have an older system Node.

## Operator Notes

- Existing non-IoT deployments are unaffected while `edge.enabled=false`.
- In IoT / Edge mode, Gateway still does not call Center on each proxied
  request. Approval checks use local cached status; Center polling and config
  snapshot sync run outside the proxy hot path.
- Config snapshots are bounded and redacted. Enrollment tokens and local device
  private keys are not stored in Center snapshots.
- `make install TARGET=linux-systemd INSTALL_ROLE=gateway` writes the supervised
  process model for Gateway installs. Center installs remain a separate
  control-plane role.
- The first migration from a legacy single-process Gateway still needs a normal
  service restart because listener ownership changes.

## Build and Development

- `tools/npm-node22.sh` is now the default `NPM` used by Makefile UI targets.
- The wrapper uses local Node.js 22.12+ and npm 10+ when available, otherwise it
  runs `node:22-alpine` through Docker.
- `make ui-build-sync` and `make center-ui-build-sync` should no longer emit the
  Node 18 engine warning on hosts that still have Node 18 installed.

## Compatibility and Migration

- DB migrations are required before running v1.2.1.
- Center config snapshot storage adds schema for latest per-device snapshots.
- Existing Gateway and Center install commands remain compatible because
  `INSTALL_ROLE` still defaults to `gateway`.
