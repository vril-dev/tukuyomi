# Remote SSH Tunnel

Remote SSH lets an operator open a short-lived SSH session to an approved Gateway without exposing an inbound Gateway port.

## Security Model

- Remote SSH is disabled by default on both Center and Gateway.
- Center requires an approved device, an enabled per-device policy, a reason, TTL, and an authenticated admin token.
- Gateway accepts pending sessions only when they are signed by the pinned Center Ed25519 signing key.
- Gateway connects outbound to Center and runs an embedded SSH server inside the `tukuyomi` process.
- Operator access uses a one-time Ed25519 public key generated by `tukuyomi remote-ssh`.
- The CLI waits for the Gateway SSH host public key through Center and writes a temporary pinned `known_hosts` file before printing the SSH command.
- Center and CLI require HTTPS for Remote SSH by default. HTTP is available only for explicit local testing flags.
- Gateway refuses to run a shell as root unless `remote_ssh.gateway.embedded_server.run_as_user` is set.
- Gateway starts shells with a minimal environment instead of inheriting the `tukuyomi` process environment.
- Port forwarding, SFTP, SCP, agent forwarding, and arbitrary SSH subsystems are not enabled.

## Configuration

Center side:

```json
{
  "edge": {
    "enabled": true
  },
  "remote_ssh": {
    "center": {
      "enabled": true,
      "max_ttl_sec": 900,
      "idle_timeout_sec": 300,
      "max_sessions_total": 16,
      "max_sessions_per_device": 1
    }
  }
}
```

Gateway side:

First fetch the Center signing public key from an authenticated Center admin session:

```sh
curl "$CENTER/center-api/remote-ssh/signing-key" \
  -H "Authorization: Bearer $TUKUYOMI_ADMIN_TOKEN"
```

Set the returned `public_key` as `remote_ssh.gateway.center_signing_public_key` on each Gateway:

```json
{
  "edge": {
    "enabled": true
  },
  "remote_ssh": {
    "gateway": {
      "enabled": true,
      "center_signing_public_key": "ed25519:REPLACE_WITH_CENTER_PUBLIC_KEY",
      "center_tls_ca_bundle_file": "conf/center-ca.pem",
      "center_tls_server_name": "center.example.local",
      "embedded_server": {
        "enabled": true,
        "shell": "/bin/sh",
        "working_dir": "/",
        "run_as_user": "tukuyomi"
      }
    }
  }
}
```

The Gateway embedded server will not start accepting Remote SSH sessions without this pinned key.

`center_tls_ca_bundle_file` is optional when Center uses a public CA/ACME certificate. For closed-network or self-signed Center certificates, put the Center CA certificate in this PEM bundle and distribute it to each Gateway. `center_tls_server_name` is optional, but useful when Gateway reaches Center by IP while the certificate is issued to a DNS name.

When bootstrapping a Center-protected Gateway, the same trust settings can be written into the Gateway config:

```sh
tukuyomi bootstrap-center-protected-gateway \
  --center-url "https://center.example.local" \
  --center-ca-bundle "conf/center-ca.pem" \
  --center-server-name "center.example.local"
```

## Operation

Enable the per-device policy from the Center API:

```sh
curl -X PUT "$CENTER/center-api/devices/$DEVICE/remote-ssh/policy" \
  -H "Authorization: Bearer $TUKUYOMI_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"enabled":true,"max_ttl_sec":900,"require_reason":true,"allowed_run_as_user":"tukuyomi"}'
```

Start a local operator tunnel:

```sh
export TUKUYOMI_ADMIN_TOKEN="$TOKEN"
tukuyomi remote-ssh \
  --center "https://center.example.com" \
  --center-ca-bundle "conf/center-ca.pem" \
  --center-server-name "center.example.local" \
  --device "$DEVICE" \
  --reason "maintenance"
```

The command prints the exact local `ssh` command. Keep `tukuyomi remote-ssh` running while the SSH session is active.

Omit `--center-ca-bundle` and `--center-server-name` when Center uses a public CA certificate and the URL hostname matches the certificate.

For local-only testing with an HTTP Center URL, add `--allow-insecure-http` or set `TUKUYOMI_REMOTE_SSH_ALLOW_INSECURE_HTTP=1`. Do not use this for production Gateways.
